# Поиск артикула по описанию

По примерному или неполному описанию товара найди точный артикул в номенклатуре или выдай список совпадений.

## Сценарии использования

- «Найди артикул по описанию: труба ПНД 32мм чёрная»
- «Что-то типа шланга ПВХ 20мм, артикул забыл»
- «Есть товар "фильтр угольный 10" какой артикул?»
- «Сколько стоит X у поставщика Y»
- «Найти товар у поставщиков»

## Приоритет источников поиска

### Поиск цен по товару/поставщику
1. **`nomenclature_mapping`** (БД `category.db`) — всегда сначала
   - Маппинг: наш артикул → поставщик → их артикул → их цена
   - Сортировка: по `confidence` (уверенность сопоставления), от большей к меньшей
   - ВСЕ кандидаты, прошедшие порог качества (нет ограничения по количеству)
2. **Если в `nomenclature_mapping` нет** → `purchase_prices` (БД `category.db`)
   - Искать по артикулу, наименованию, поставщику
3. **Если и там нет** → файлы:
   - `data/price-registry/price-registry.xlsx`
   - `purchase-prices/` — CSV из писем
4. **Если нигде нет** → «информация не найдена»

### Поиск товара по описанию
1. `nomenclature_mapping` — сопоставленные товары с ценами поставщиков (через `name_clean`)
2. `nomenclature` — наша номенклатура
3. `price_registry` — реестр прайсов поставщиков
4. Файлы при необходимости

## Алгоритм поиска при запросе "Найти товар у поставщиков"

1. Сделать поиск по **`nomenclature_mapping`** (по названию товара из нашей номенклатуры или по описанию)
2. **Если результатов > 10 SKU** → уточнить у пользователя:
   - Точное название
   - Объём, вес, фасовка
   - Другие детали для сужения поиска
3. **Если результатов ≤ 10 SKU** → вывести таблицу в строго согласованном формате
4. **Если ничего не найдено** → перейти к поиску в `purchase_prices`
5. Если и там пусто → «информация не найдена»

## Обязательная проверка закупочных цен

**Перед каждым выводом результатов — всегда проверять `purchase_prices`:**
- Запрос: `SELECT price_with_vat, supplier FROM purchase_prices WHERE article = ? AND price_with_vat IS NOT NULL ORDER BY date(document) DESC LIMIT 1`
- Артикул брать из первого товара в результатах
- Если данные есть — выводить строку «Последняя закупочная цена (с НДС): **сумма**» и «Поставщик: »
- Проверять тщательно, по точному артикулу
- Если данных нет — писать «Последняя закупочная цена: нет данных»

## Строгий формат выдачи (утверждён 23.06.2026)

```
Найдены товары по запросу: **текст запроса пользователя**

Последняя закупочная цена (с НДС): **сумма.00₽**
Поставщик: Полное название поставщика

| Поставщик | Наименование у поставщика | Цена,₽ | Разница | Совпадение |
|---|---|---|---|---|
| Полное название поставщика | Название товара из прайса | 0 000.00 | +0.0% | 00% |
| ... | ... | ... | ... | ... |

_...и ещё N позиций от Поставщик1, Поставщик2_
```

**Правила выдачи:**
- Заголовок: `Найдены товары по запросу:` — обычный текст, запрос **жирным**
- Пустая строка после заголовка
- Последняя закупочная цена (с НДС) — из `purchase_prices`, самое новое по дате, **жирным**
- Строка «Поставщик: » — от кого эта закупочная цена
- Пустая строка после поставщика
- Закупочная цена и поставщик всегда между заголовком и таблицей

**Правила таблицы:**
- Колонки строго: Поставщик | Наименование у поставщика | Цена,₽ | Разница | Совпадение
- «Поставщик» — полное название как в БД
- «Наименование у поставщика» — поле `supplier_name` из `nomenclature_mapping`, коротко (обрезать если длиннее 50 символов)
- «Цена,₽» — из `nomenclature_mapping.price`, разделитель тысяч — пробел
- «Разница» — ((цена поставщика − закупочная цена) / закупочная цена × 100), формат `+0.0%` или `−0.0%`
- «Совпадение» — из `nomenclature_mapping.confidence`, формат `00%`
- Никаких переносов строк — каждая строка таблицы целиком на одной строке
- Сортировка: по Совпадение (confidence) от большего к меньшему
- Дубли поставщиков удалять (одинаковый поставщик + одинаковое наименование — одна строка)
- Если нет закупочной цены — строка «Последняя закупочная цена: нет данных» и колонка Разница = «—»
- Если нет данных в `nomenclature_mapping` — переход к `purchase_prices`

**Итоговая строка:**
- После таблицы — строка вида: `...и ещё N позиций от Поставщик1, Поставщик2`
- N = общее количество выведенных строк в таблице
- Поставщики — перечислить уникальных через запятую
- Если строк в таблице 0 — не выводить

## Источники номенклатуры

- Выгрузка из 1С через `nomenclature` (БД)
- `nomenclature_mapping` (БД) — сопоставление с поставщиками
- `price_registry` (БД) — реестр прайсов поставщиков
- `purchase_prices` (БД) — закупочные цены
- Ранее обработанные прайсы
