# Извлечение данных из переписки

Автоматический мониторинг входящих писем от поставщиков на предмет изменений: смена менеджера, реквизитов, контактов, банка.

## Сценарии использования

- Автоматически: пришло письмо от Поставщика Х, в нём сообщение о новом менеджере
- «Проверь последние письма от поставщиков на предмет изменений»

## Распознаваемые события

| Событие | Примеры в письме |
|---------|-----------------|
| Смена менеджера | «новый менеджер», «уволен», «передаю дела», «теперь со мной», «представляю...» |
| Смена банка/реквизитов | «изменились реквизиты», «новый РС», «смена банка», «новые платёжные данные» |
| Смена адреса/контактов | «новый адрес», «изменили номер/email», «переезд офиса» |
| Смена юрлица | «переименование», «реорганизация», «теперь ООО» |

## Действия

1. При получении письма от известного поставщика — проанализировать текст
2. Если обнаружено событие — извлечь:
   — Тип события
   — Старое значение (если указано)
   — Новое значение
   — Дата вступления (если указана)
3. Обновить карточку поставщика в базе знаний (навык #27)
4. Уведомить пользователя:
   «У Поставщика Х сменился менеджер: было [старый → новый]»
5. Сохранить запись в истории изменений поставщика

## Техника обработки

- Для классификации письма (событие/не событие) — короткий промпт через Ollama
- Для извлечения деталей — DeepSeek
- Периодический запуск: каждый раз при проверке почты
