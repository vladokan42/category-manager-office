# Анализ сбоев и ротация поставщиков

При получении письма от поставщика о проблемах (перебои, дефицит, снятие с производства, задержки) — проанализировать, найти альтернативы, предложить ротацию.

## Сценарии использования

- «Пришло письмо от Поставщика Х — у них проблемы с производством. Что делать?»
- «Поставщик Y снял с производства позицию Z, найди замену»
- «Поставщик W задерживает отгрузки, предложи альтернативу»

## Классификация проблем

| Тип | Действие |
|-----|----------|
| Временный перебой (известна дата возобновления) | Мониторинг, предложить запасной вариант на период |
| Снятие с производства / дефицит | Срочная ротация, поиск нового поставщика |
| Повышение цен | Передать в навык `price-increase-analysis` (#5) |
| Систематические задержки | Предложить альтернативного поставщика |

## Действия

1. Извлечь из письма:
   — Поставщик, тип проблемы, список затронутых позиций
   — Сроки (когда возобновится / когда снимают / срок задержки)
2. По каждой затронутой позиции:
   — Найти альтернативных поставщиков (из базы знаний #27)
   — Сравнить цены (из журнала #24 / базы)
   — Оценить: есть ли аналог, какая цена, условия
3. Сформировать отчёт:
   ```
   Ситуация: [что случилось]
   Затронуто: [список позиций]
   
   Варианты замены:
   — [Поставщик А]: [цена], [условия] → 🔥 рекомендую
   — [Поставщик Б]: [цена], [условия] → запасной
   
   Рекомендация: [переключить / подождать / искать нового]
   ```
