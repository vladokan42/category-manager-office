# Формат вывода цен

Сотрудники имеют тот же уровень доступа, что и Господин. Формат — такой же, как у скилла `article-search` основного workspace.

## Базовый шаблон

```
Найдены товары по запросу: **запрос пользователя**

Последняя закупочная цена (с НДС): **сумма.00₽**
Поставщик: Полное название поставщика

| Поставщик | Наименование у поставщика | Цена,₽ | Разница | Совпадение |
|---|---|---|---|---|
| ... | ... | ... | ... | ... |
```

## Сравнительная таблица (от нескольких поставщиков)

```
Поставщик                    | Артикул       | Описание                     | Реестр,₽ | Разн,₽ | Дата
```

- Сортировка по Разн,₽ (отрицательные → положительные)
- В моноширном код-блоке
- Знак рубля ₽

## Правила

- Указываешь дату прайса
- Если данных нет — «нет данных» прямо, не выдумываешь
- НДС: если в запросе не указан — показывай с НДС (по умолчанию)
- Округление до копеек
- Если у поставщика есть пометки (МРЦ, минимальная партия, бонусы) — добавляешь после таблицы

## Чего НЕ делать

- Не предлагать «отправить запрос поставщику» — это запрещено
- Не говорить про твою личную закупочную стратегию или планы Господина по ротации
- Не делать пометок «это конфиденциально, никому не показывай» — у всех уровень доступа равный
